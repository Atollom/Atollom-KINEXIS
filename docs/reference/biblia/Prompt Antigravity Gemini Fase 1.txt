2.2 CHECKPOINT_LOAD — Ejecutar al inicio de cada sesión de relevo
## AL INICIAR UNA SESIÓN DE RELEVO — LEER ESTO PRIMERO:

CHECKPOINT_LOAD {
  1. Leer SHARED_MEMORY.md completo
  2. Leer el último CHECKPOINT_SAVE en el historial de commits
  3. Abrir el archivo indicado en stopped_at.file
  4. Ir a la línea stopped_at.line
  5. Ejecutar: git status + git log --oneline -5
  6. Ejecutar: pytest tests/ para ver qué está passing/failing
  7. Reportar status antes de escribir una sola línea de código:

  LOAD_REPORT: {
    loaded_from: 'CHECKPOINT session N por GEMINI/CLAUDE'
    understood_context: true/false
    tests_status: 'N passing, M failing'
    ready_to_continue: true/false
    clarification_needed: ['pregunta específica si algo no está claro']
    first_action: 'exactamente qué va a hacer primero'
  }
}
